# Habit Streak (habit-streak)

# PRD
## Background / Problem
- **데이터 포인트(브리프 기반):** 타깃 사용자는 **20–40대**이며, MVP 성공 지표로 **DAU 1,000명**, **7일 리텐션 40%+**, **평균 세션 2분+**가 제시되어 있다.  
사람들은 좋은 습관을 시작하지만 꾸준히 유지하기 어렵다. 기존 습관 앱들은 기능이 복잡해 “기록 자체”가 부담이 되어 중도 포기를 유발한다. 따라서 **매일 빠르게 체크하고, 스트릭과 달성률을 즉시 확인**할 수 있는 단순한 습관 트래커가 필요하다.

## Goal (1-sentence product definition)
사용자가 **오늘의 체크리스트에서 10초 내 습관 완료 체크**를 끝내고, **주/월 달성률과 스트릭을 확인**하며, 출시 후 **DAU 1,000명 및 7일 리텐션 40%**를 달성한다.

## Non-goals
- 소셜 기능(친구, 공유, 랭킹)
- 푸시 알림(리마인더 포함)
- 클라우드 동기화/멀티 디바이스 동기화(로컬 저장소만)
- 인앱 결제/구독 결제 연동(스토어 결제 포함) 및 외부 결제(toss/kakaopay/stripe) 연동
- 고급 목표/코칭(챌린지, AI 추천, 루틴 자동 생성)

## Target Users (personas + use cases)
1. **김민지(29세), 직장인**
   - **상황/목표:** 운동/독서 습관을 만들고 싶지만 매번 기록 앱이 번거로워 금방 포기함
   - **페인 포인트:** 앱 진입 후 “오늘 뭐 해야 하지?”가 복잡하면 이탈
   - **사용 사례:** 출근길에 앱을 열고 오늘의 습관을 빠르게 체크, 스트릭을 보며 동기 부여

2. **박준호(38세), 자영업자**
   - **상황/목표:** 수면/영양제/정리 같은 생활 습관을 꾸준히 관리하고 싶음
   - **페인 포인트:** 며칠 빠지면 전체 기록을 보기 싫어지고 재시작이 어려움
   - **사용 사례:** 캘린더에서 놓친 날짜를 확인하고, 주/월 통계로 “유지 중인 습관”을 파악

## Target Market
- **지역:** 대한민국(South Korea)
- **시장:** 습관 형성/자기관리(간편 체크리스트형 트래커 선호층)
- **포지셔닝:** 복잡한 기능을 덜어낸 “하루 체크 + 스트릭 + 통계” 중심의 라이트 습관 앱

## Data Entities (nouns with key fields)
> (User는 템플릿에 이미 존재하므로 제외)

1. **Habit**
   - `id` (string, uuid)
   - `name` (string)
   - `icon` (string; 예: 이모지 또는 프리셋 키)
   - `categoryId` (string | null)
   - `frequencyType` (enum: `daily` | `weekly`)
   - `weeklyTarget` (number | null; 예: 주 3회)
   - `isArchived` (boolean)
   - `createdAt` (datetime)
   - `sortOrder` (number)

2. **Category**
   - `id` (string, uuid)
   - `name` (string)
   - `color` (string; hex)

3. **CompletionRecord**
   - `id` (string, uuid)
   - `habitId` (string)
   - `date` (string; `YYYY-MM-DD`, 로컬 기준)
   - `completed` (boolean)
   - `completedAt` (datetime | null)

4. **StreakState** *(계산 캐시 또는 매번 계산 가능 — 구현 선택)*
   - `habitId` (string)
   - `currentStreak` (number)
   - `bestStreak` (number)
   - `lastCompletedDate` (string | null)

## Core Flow (numbered steps)
1. 사용자가 앱을 열면 **오늘의 습관 체크리스트**(카테고리 그룹핑 포함)를 본다.
2. 사용자는 습관 항목을 **탭**하여 오늘 완료 처리한다(즉시 UI 반영).
3. 완료 처리와 동시에 해당 습관의 **스트릭(연속 달성일)**이 갱신되어 표시된다.
4. 사용자는 하단 탭에서 **통계**로 이동해 **주간/월간 달성률**을 확인한다.
5. 사용자는 하단 탭(또는 화면 내 +)에서 **새 습관 추가**로 이동해 이름/아이콘/빈도(매일, 주 N회)/카테고리를 설정하고 저장한다.
6. 사용자는 **캘린더 뷰**에서 특정 날짜를 선택해 해당 날짜의 달성 기록(완료/미완료)을 확인한다.

## Success Metrics (measurable)
- **DAU:** 1,000명 달성
- **7일 리텐션:** 40% 이상
- **평균 세션 시간:** 2분 이상
- **체크 완료 전환율:** 앱 실행 세션 중 “최소 1개 습관 체크” 발생 비율 60% 이상 *(측정 이벤트 정의 필요)*
- **주간 재방문 빈도:** 주 3일 이상 앱 오픈 사용자 비율 35% 이상

## MVP Scope (exhaustive feature list)
> 템플릿 제공 항목(인증/탭/기본 컴포넌트/secure store/zustand/api client)은 제외하고 “고유 기능”만 기재

- **습관 생성 및 관리**
  - 이름, 아이콘, 빈도(매일/주 N회), 카테고리 지정, 아카이브(숨김) 처리
- **오늘의 습관 체크리스트 화면**
  - 오늘 기준 해야 할 습관 노출, 탭으로 완료 토글, 카테고리별 그룹핑 표시
- **스트릭(연속 달성일) 표시**
  - 습관별 현재 스트릭/최고 스트릭 표시(정의: 연속 일자 기준)
- **캘린더 뷰**
  - 날짜별 완료 여부 시각화, 특정 날짜 선택 시 기록 상세 확인
- **주간/월간 통계 대시보드**
  - 기간별 달성률(%) 및 완료 횟수 요약(주간/월간 전환)
- **오프라인 로컬 저장(offline-first)**
  - 네트워크 없이도 생성/체크/조회 가능(클라우드 동기화는 제외)

### 모바일 상호작용/UX 명세(MVP)
- **Pull-to-refresh 적용 화면**
  - 오늘 체크리스트: 아래로 당겨 **오늘 데이터/계산값(스트릭/달성률) 재계산 후 즉시 반영**
  - 통계 대시보드: 아래로 당겨 **주/월 집계 재계산**
  - 캘린더 뷰: 아래로 당겨 **월별 마킹/선택일 상세 재로딩**
- **제스처**
  - 습관 관리 리스트에서 항목 **스와이프 → 아카이브(숨김) 또는 삭제(확인 다이얼로그 포함)**  
    - 패스 조건: 스와이프 액션 후 확인을 누르면 해당 습관이 리스트에서 즉시 제거(삭제) 또는 숨김(아카이브) 처리
  - 습관 항목 **롱프레스 → 편집 화면으로 이동**
    - 패스 조건: 롱프레스 시 300ms 내 편집 화면 네비게이션 발생
- **카메라/갤러리**
  - MVP에서는 **미사용**(아이콘은 이모지/프리셋 선택으로 제한)

## Target Audience & Marketing
- **핵심 타깃:** 20–40대, “복잡한 기능 없이 체크리스트로만 관리”를 원하는 사용자
- **메시지:** “하루 10초 체크, 스트릭과 달성률로 꾸준함 유지”
- **채널(브리프 기반):** KakaoTalk, Naver Blog, YouTube Korea  
  - 예: 유튜브 쇼츠로 “오늘 체크 3초” 데모, 네이버 블로그에 주간 달성률 화면 중심 후기, 카카오톡 채널로 업데이트 공지(푸시 아님)

## Monetization Strategy
- **freemium**  
  - 기본 기능(습관 체크/스트릭/캘린더/통계/카테고리)은 무료 제공, 향후 프리미엄(예: 더 많은 습관 수, 고급 통계)을 **App Store / Google Play 인앱 결제**로 제공하는 방향 권장.  
  - *참고:* 브리프에 toss/kakaopay/stripe가 있으나, 모바일 앱 결제는 스토어 결제를 우선하므로 MVP에서는 결제 연동을 진행하지 않는다.

## Assumptions
1. 사용자는 “오늘 할 일”이 첫 화면에 보이면 앱 재방문 확률이 높아진다.
2. 체크 동작은 **탭 1회**로 끝나야 한다(추가 입력 최소화).
3. 빈도는 MVP에서 **매일/주 N회**만으로도 충분하다(복잡한 규칙은 제외).
4. 스트릭은 습관 유지 동기 부여에 핵심이며, 즉시 갱신 표시가 필요하다.
5. 클라우드 동기화 없이도 초기 MVP 검증(DAU/리텐션)은 가능하다.
6. 로컬 날짜 기준(`YYYY-MM-DD`)으로 기록하면 타임존 이슈를 MVP에서 최소화할 수 있다.
7. 카테고리 그룹핑은 목록 복잡도를 줄여 “체크” 중심 UX를 돕는다.
8. 통계는 고급 차트보다 **달성률/완료 횟수 요약**이 MVP에 적합하다.

2. 습관 삭제 정책: 삭제 시 과거 기록(CompletionRecord)도 함께 삭제할지, 익명화/보존할지?
3. 아카이브(숨김)와 삭제를 모두 제공할지, MVP에서는 아카이브만 제공할지?
4. 아이콘 선택 방식: 이모지 입력 허용 vs 프리셋 목록만 제공 중 어느 쪽이 MVP에 적합한가?
5. 프리미엄에서 제한할 요소(습관 개수, 카테고리 수, 통계 기간 등)는 무엇이 가장 설득력 있는가?
